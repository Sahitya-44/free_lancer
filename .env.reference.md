# Environment Variables Reference

## Backend Configuration (.env)

### Required Variables

```env
# Database Configuration
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/freelancer_db

# JWT Configuration
JWT_SECRET=your_super_secret_jwt_key_min_32_chars_change_in_production
JWT_EXPIRE=7d

# Server Configuration
PORT=5000
NODE_ENV=development

# Frontend URL (for CORS)
FRONTEND_URL=http://localhost:3000
```

### Optional Variables

```env
# Email Configuration (for future use)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_password

# File Upload
MAX_FILE_SIZE=10485760
UPLOAD_PATH=./uploads

# Rate Limiting
RATE_LIMIT_WINDOW=15m
RATE_LIMIT_MAX_REQUESTS=100
```

## Frontend Configuration (.env)

### Required Variables

```env
# API Configuration
REACT_APP_API_URL=http://localhost:5000/api

# App Title
REACT_APP_TITLE=PortfolioHub
```

### Optional Variables

```env
# Analytics
REACT_APP_ANALYTICS_ID=your_analytics_id

# Third-party APIs
REACT_APP_CLOUDINARY_URL=your_cloudinary_url
```

## How to Generate Secure JWT Secret

### Using OpenSSL (Linux/Mac)
```bash
openssl rand -hex 32
```

### Using Node.js
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Using Python
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

## MongoDB Connection Strings

### Local MongoDB
```env

```

### MongoDB Atlas (Cloud)
```env
MONGODB_URI=mongodb+srv://username:password@cluster-name.mongodb.net/freelancer_db?retryWrites=true&w=majority
```

**Note:** Replace `username`, `password`, and `cluster-name` with your actual values.

## Environment Variable Validation

### Checklist

- [ ] All required variables are set
- [ ] Database connection is valid
- [ ] JWT secret is strong (min 32 characters)
- [ ] PORT is not in use
- [ ] FRONTEND_URL matches actual frontend URL
- [ ] API_URL in frontend matches backend URL
- [ ] NODE_ENV matches deployment environment
- [ ] No sensitive data in version control

## Common Issues

### MongoDB Connection Failed

**Cause:** Invalid connection string or database service not running

**Solution:**
1. Check MongoDB service is running
2. Verify connection string syntax
3. Check username and password
4. If using Atlas, whitelist your IP

### JWT Token Invalid

**Cause:** JWT_SECRET mismatch between encoding and decoding

**Solution:**
1. Use same JWT_SECRET across all instances
2. Don't change JWT_SECRET without re-authenticating users
3. Generate new secret using recommended methods

### CORS Error

**Cause:** FRONTEND_URL doesn't match actual frontend URL

**Solution:**
1. Update FRONTEND_URL to exact frontend URL
2. Include protocol (http/https)
3. Include port if not default (80/443)
4. Restart backend server

### Port Already in Use

**Cause:** Another process using the PORT

**Solution:**
1. Change PORT to different value (e.g., 5001)
2. Or kill process using the port
3. Or use process manager (PM2)

## Security Best Practices

✅ **Do:**
- Use strong JWT secret (min 32 characters)
- Never commit .env to version control
- Use different secrets for different environments
- Rotate secrets periodically
- Use HTTPS in production
- Whitelist trusted origins only

❌ **Don't:**
- Use simple secrets like "secret" or "password"
- Share .env file in emails or messages
- Use same secret in dev and production
- Commit .env to git
- Expose .env on public servers
- Use default database passwords
